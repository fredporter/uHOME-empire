# uHOME-empire Boundary

## Owns

- publishing systems and operational packs
- campaign and messaging workflow packaging
- audience segmentation and contact-operation helpers
- educational make-and-deploy patterns for inspectable workflow systems
- vault-first local-network sync orchestration for configured user vaults
- Google Docs mirror sync and Google Keep or Tasks projection
- HubSpot contact, task, and CRM organization using app-owned contact records
  plus local enrichment
- always-on webhook-style sync actions and console workflow surfaces
- sync-record profile ownership for empire-specific queue and routing behavior
- queueable sync-package handoff into the `uHOME-server` local-network ingest
  surface
- remote job and container definitions for webhook, Google, and HubSpot lanes

## Does Not Own

- Apple or iCloud sync, AppleScript automation, and native macOS handling
- canonical user or binder contact database ownership
- canonical runtime semantics or source-of-truth content contracts
- family-owned sync-record schema ownership
- the main family AI assistance layer
- canonical web publishing and email rendering contracts owned by the shared
  public render/theme pipeline
- generic provider transport infrastructure owned by `uDOS-wizard`
- base runtime hosting or local execution ownership owned by `uHOME-server`
